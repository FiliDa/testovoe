## 🚀 Возможности

- **Управление задачами** - полный CRUD для задач медицинского персонала
- **Периодичность выполнения** - 4 типа автоматического повторения задач
- **Безопасность** - многоуровневая защита от уязвимостей
- **REST API** - современный JSON API с документацией Swagger
- **Контейнеризация** - готовность к развертыванию через Docker

- Go 1.23+


После запуска система будет доступна по адресу: `http://localhost:8080`

### Документация API

- **Swagger UI**: http://localhost:8080/swagger/
- **OpenAPI JSON**: http://localhost:8080/swagger/openapi.json

## 🔄 Типы периодичности задач

Система поддерживает 4 типа автоматического повторения задач:

### 1. Ежедневные задачи
Повторение каждые N дней

```json
{
  "periodicity_type": "daily",
  "periodicity_interval": 3
}
```

### 2. Ежемесячные задачи
Выполнение в конкретные дни месяца

```json
{
  "periodicity_type": "monthly",
  "periodicity_days": [1, 15, 28]
}
```

### 3. Конкретные даты
Выполнение только в указанные даты

```json
{
  "periodicity_type": "specific_dates",
  "periodicity_dates": ["2024-03-15", "2024-06-15"]
}
```

### 4. Четные/нечетные дни
Автоматический расчет подходящих дней

```json
{
  "periodicity_type": "even_odd",
  "periodicity_even_odd": "even"
}
```

## 📡 API документация

Базовый префикс API: `/api/v1`

### Основные endpoint'ы

| Метод | Endpoint | Описание |
|-------|----------|-----------|
| POST | `/tasks` | Создание новой задачи |
| GET | `/tasks` | Получение списка всех задач |
| GET | `/tasks/{id}` | Получение задачи по ID |
| PUT | `/tasks/{id}` | Обновление задачи |
| DELETE | `/tasks/{id}` | Удаление задачи |

### Примеры запросов

Создание задачи с ежедневной периодичностью:

```bash
curl -X POST "http://localhost:8080/api/v1/tasks" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Ежедневный обзвон пациентов",
    "description": "Обзвонить пациентов для подтверждения визита",
    "status": "new",
    "periodicity_type": "daily",
    "periodicity_interval": 3
  }'
```

Полный список примеров доступен в файле [EXAMPLES.md](EXAMPLES.md).

## 🛡️ Безопасность

### Многоуровневая защита

1. **Rate Limiting** - 100 запросов в минуту на IP адрес
2. **Security Headers** - защита от XSS и других веб-уязвимостей
3. **Валидация входных данных** - предотвращение инъекций
4. **Параметризованные запросы** - защита от SQL-инъекций

### Заголовки безопасности

Система автоматически добавляет security headers:
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`
- `Content-Security-Policy`
- `Strict-Transport-Security`
