# Тестовые сценарии для системы периодичности

## 1. Тестирование валидации данных

### 1.1. Корректные данные
```json
{
  "title": "Тестовая задача",
  "periodicity_type": "daily",
  "periodicity_interval": 5
}
```
**Ожидаемый результат:** Успешное создание задачи

### 1.2. Некорректный тип периодичности
```json
{
  "title": "Тестовая задача",
  "periodicity_type": "invalid_type"
}
```
**Ожидаемый результат:** Ошибка валидации "invalid periodicity type"

### 1.3. Отрицательный интервал
```json
{
  "title": "Тестовая задача", 
  "periodicity_type": "daily",
  "periodicity_interval": -1
}
```
**Ожидаемый результат:** Ошибка валидации "daily periodicity requires positive interval"

### 1.4. Некорректные дни месяца
```json
{
  "title": "Тестовая задача",
  "periodicity_type": "monthly", 
  "periodicity_days": [0, 32]
}
```
**Ожидаемый результат:** Ошибка валидации "monthly day must be between 1 and 31"

## 2. Тестирование вычисления следующих дат

### 2.1. Ежедневная периодичность
**Текущая дата:** 2024-01-01
**Интервал:** 3 дня
**Ожидаемая следующая дата:** 2024-01-04

### 2.2. Ежемесячная периодичность 
**Текущая дата:** 2024-01-10
**Дни месяца:** [1, 15, 20]
**Ожидаемая следующая дата:** 2024-01-15

### 2.3. Ежемесячная периодичность (переход на следующий месяц)
**Текущая дата:** 2024-01-25
**Дни месяца:** [1, 5]
**Ожидаемая следующая дата:** 2024-02-01

### 2.4. Конкретные даты
**Текущая дата:** 2024-01-01
**Даты:** ["2024-03-15", "2024-06-15", "2024-09-15"]
**Ожидаемая следующая дата:** 2024-03-15

### 2.5. Четные дни
**Текущая дата:** 2024-01-01 (нечетный день)
**Тип:** even
**Ожидаемая следующая дата:** 2024-01-02

### 2.6. Нечетные дни  
**Текущая дата:** 2024-01-02 (четный день)
**Тип:** odd
**Ожидаемая следующая дата:** 2024-01-03

## 3. Тестирование безопасности

### 3.1. SQL-инъекция в названии задачи
```json
{
  "title": "Тест'; DROP TABLE tasks; --",
  "periodicity_type": "daily",
  "periodicity_interval": 1
}
```
**Ожидаемый результат:** Задача создается с экранированным названием

### 3.2. XSS-инъекция в описании
```json
{
  "title": "Тестовая задача",
  "description": "<script>alert('xss')</script>",
  "periodicity_type": "daily", 
  "periodicity_interval": 1
}
```
**Ожидаемый результат:** Задача создается с экранированным описанием

### 3.3. Rate limiting
**Действие:** 101 запрос за 1 минуту
**Ожидаемый результат:** 429 Too Many Requests после 100 запросов

## 4. Интеграционное тестирование

### 4.1. Полный цикл создания задачи
1. Создать задачу с периодичностью
2. Получить задачу по ID
3. Проверить что next_occurrence вычислен корректно
4. Обновить задачу
5. Проверить что next_occurrence пересчитан

### 4.2. Смешанные типы задач
1. Создать несколько задач разных типов периодичности
2. Получить список всех задач
3. Проверить что все задачи возвращаются корректно
4. Проверить что next_occurrence вычислен для периодических задач

### 4.3. Обновление периодичности
1. Создать задачу с daily периодичностью
2. Обновить на monthly периодичность
3. Проверить что next_occurrence пересчитан корректно

## 5. Пограничные случаи

### 5.1. Февраль и 31-е число
**Текущая дата:** 2024-01-31
**Дни месяца:** [31]
**Ожидаемая следующая дата:** 2024-03-31 (пропускаем февраль)

### 5.2. Високосный год
**Текущая дата:** 2024-02-28
**Дни месяца:** [29]
**Ожидаемая следующая дата:** 2024-02-29

### 5.3. Невисокосный год  
**Текущая дата:** 2023-02-28
**Дни месяца:** [29]
**Ожидаемая следующая дата:** 2023-03-29

### 5.4. Пустые массивы данных
```json
{
  "title": "Тестовая задача",
  "periodicity_type": "monthly",
  "periodicity_days": []
}
```
**Ожидаемый результат:** Ошибка валидации "monthly periodicity requires at least one day"

### 5.5. Прошедшие даты
```json
{
  "title": "Тестовая задача", 
  "periodicity_type": "specific_dates",
  "periodicity_dates": ["2023-12-01"]
}
```
**Ожидаемый результат:** Ошибка валидации "specific dates must be in the future"
